REST API Reference

This document describes the User Management API endpoints.

Authentication

All requests require a Bearer token in the Authorization header.

Authorization: Bearer <your-api-key>

Endpoints

GET /api/users

Returns a list of all users. Supports pagination via page and limit query parameters.

  • Default page size: 20
  • Maximum page size: 100
  • Sort options: name, created_at, email

POST /api/users

Creates a new user account.

FieldTypeRequiredDescription
namestringYesFull name of the user
emailstringYesEmail address (must be unique)
rolestringNoOne of: admin, editor, viewer
activebooleanNoAccount status (default: true)

DELETE /api/users/:id

Deletes a user by ID. Returns 204 No Content on success.

Rate Limits

API calls are limited to 1000 requests per hour. Exceeding this limit returns a 429 status code.
